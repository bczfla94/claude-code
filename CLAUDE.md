## Git Workflow

After every meaningful change, commit locally with a clean, descriptive commit message and push to GitHub:

```bash
git add <files>
git commit -m "Short description of what changed and why"
git push
```

- Always push to `origin/main` after committing
- Commit messages should be concise and describe the intent, not just the mechanics
- Never skip commits — the goal is to always have a recoverable version on GitHub
